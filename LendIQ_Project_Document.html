System Architecture

2.1 High-Level Flow

A loan application travels through four distinct layers before a decision is returned:

  1. Ingestion — Java API Gateway validates, rate-limits, and publishes to Kafka after a Redis fraud pre-check (sliding window velocity).
  2. Scoring — Three parallel workers consume the Kafka topic: a custom Decision Tree (Java), a LightGBM ML service (Python gRPC), and an async GNN fraud detector (Python).
  3. Routing — Java Score Aggregator combines all scores, runs the Interval Tree lender matcher, and picks the best lender from a Red-Black Tree sorted queue.
  4. Storage — Every decision is appended to PostgreSQL (audit), documents stored in S3, velocity counters maintained in Redis.

2.3 Key Design Decisions

Kafka partitioned by applicant_id

All messages from the same applicant always hit the same partition and consumer. Stateful stream processing (sliding window deque) never needs distributed coordination — it lives in the consumer's local memory.

ONNX in-process inference

The LightGBM model is exported to ONNX format and loaded directly into the Python gRPC service's memory. There is no secondary network hop to a model server. This is what makes p99 < 8ms achievable for ML scoring.

Async GNN fraud check

The GNN runs on a separate Kafka consumer group and publishes its result to a fraud-results topic. The Score Aggregator waits up to 50ms for this result — meaning GNN detection never adds more than 50ms to the critical path.

Immutable audit log

The decisions table is append-only — no UPDATE or DELETE is ever issued. Regulators require a tamper-proof record of every credit decision including model version, input features, SHAP values, and the lender who received the referral.

DSA Implementations

All five data structures below are implemented from scratch in Java. This deliberately avoids TreeMap, SortedSet, or any java.util substitute — the point is to demonstrate the implementation, not the import.

Interval Tree

O(log n + k)
Where: Lender eligibility matching
Why this one: Handles overlapping income/age ranges — no other structure does this natively. A plain BST can't answer "which intervals contain point X" efficiently.

Red-Black Tree

O(log n)
Where: Sorted lender queue by match score
Why this one: Self-balancing guarantees worst-case O(log n) unlike a plain BST that degrades to O(n) on sorted insertions.

Decision Tree

O(depth)
Where: Interpretable credit scoring
Why this one: Regulators require explainability — ML black-boxes cannot be the sole decision maker. SHAP values stored at each leaf.

Sliding Window (Deque)

O(1) amortised
Where: Velocity fraud detection in Redis
Why this one: Fixed memory regardless of traffic volume — a sorted set of timestamps with O(1) eviction of expired entries.

3.1 Interval Tree — Lender Eligibility Matching

The problem

Each lender has eligibility ranges: income 40K–120K, age 25–60, min score 650. These are intervals. Given an applicant's profile, you need all lenders whose intervals contain the applicant's values — this is the stabbing query problem.

Implementation sketch (Java)

class IntervalNode {
    double low, high, maxHigh;  // income range + subtree max
    LenderRule rule;
    IntervalNode left, right;
}

List<LenderRule> query(double point) {
    List<LenderRule> results = new ArrayList<>();
    queryHelper(root, point, results);
    return results;
}

void queryHelper(IntervalNode node, double point, List<LenderRule> out) {
    if (node == null || node.maxHigh < point) return;
    if (node.low <= point && point <= node.high) out.add(node.rule);
    queryHelper(node.left, point, out);
    queryHelper(node.right, point, out);
}

At startup, all ~1000 lender rules are bulk-loaded. The tree is rebuilt every 5 minutes from the lenders table via an Airflow task — zero downtime via atomic reference swap.

3.2 Red-Black Tree — Sorted Lender Queue

What it does

After the Interval Tree returns eligible lenders, each is assigned a composite match score: 70% credit score compatibility + 20% lender acceptance rate history + 10% fee competitiveness. The Red-Black Tree keeps this list sorted so the best-fit lender is always at the top in O(log n).

  • insert(lender, matchScore) — O(log n), triggers rebalancing rotations if needed
  • getMax() — O(log n), returns lender with highest match score
  • delete(lenderId) — O(log n), used when a lender is temporarily paused
  • Rotation invariants: black-height equal on all paths, no consecutive red nodes

3.4 Sliding Window — Velocity Fraud Detection

If the same IP address or device fingerprint submits more than 5 applications within a 60-minute window, the request is flagged before it even enters Kafka — saving ML inference cost on obvious fraud.

// Redis key: velocity:{ip_hash}:{device_fp}
// Value: sorted set of timestamps

boolean isVelocityFraud(String key, Instant now) {
    long windowStart = now.minusSeconds(3600).toEpochMilli();
    redisCommands.zremrangebyscore(key, 0, windowStart); // evict old
    long count = redisCommands.zcard(key);
    redisCommands.zadd(key, now.toEpochMilli(), UUID.randomUUID().toString());
    redisCommands.expire(key, 3600);
    return count >= VELOCITY_THRESHOLD; // threshold = 5
}
04

ML Pipeline

Three models run in parallel on each application. Their outputs are combined by the Score Aggregator using a weighted blend. Each model is versioned, shadow-tested before promotion, and retrained weekly via Airflow.

4.1 Model 1 — LightGBM Credit Scorer

Model TypeLightGBM Gradient Boosted Trees (regression + binary outputs)
Training DataHistorical loan applications with 24-month repayment outcomes — 500K records for initial training
OutputCredit score 0–1000 + probability of default (0–1) + SHAP feature importances
ServingONNX Runtime in-process within Python gRPC service — no separate model server, no extra network hop
Performancep99 inference < 8ms, AUC-ROC > 0.82 on holdout set
Retraining TriggerPSI (Population Stability Index) > 0.2 on any top-5 feature, or weekly scheduled regardless

Key engineered features

  • DTI ratio: total_monthly_debt / monthly_income
  • Income stability: income_std_6months / income_mean
  • Credit utilisation: current_balance / credit_limit
  • Delinquency recency: months since last missed payment
  • Loan purpose risk bucket (home=1 → business=4)
  • Device trust score from velocity fraud pre-check result

4.2 Model 2 — PyTorch GNN Fraud Detector

Model TypeGraph Attention Network (GAT) — 3 layers, 8 attention heads
Graph StructureNodes: applicants, devices, IPs, postal codes. Edges: shared attributes (same device = edge, same IP subnet = edge, same address = edge)
OutputFraud probability (0–1) per applicant node + fraud ring ID if detected
Why GNNTraditional ML sees applicants in isolation. GNN sees the network — 20 applicants sharing one device across 3 days is invisible to tabular ML but trivial for GNN
Async ExecutionRuns on separate Kafka consumer group. Score Aggregator waits max 50ms then proceeds with fraud_prob=null if timeout

4.3 Model 3 — Fairness Layer (AIF360)

LibraryIBM AI Fairness 360 (AIF360) — open source fairness toolkit
Fairness MetricDemographic parity difference: |P(approve|group A) - P(approve|group B)| < 0.10
Protected AttributesGender, age group (18-25, 26-40, 41-60, 60+), geography tier (metro/tier-2/tier-3)
MitigationReweighting training samples (pre-processing) + adversarial debiasing (in-processing)
Reject InferenceSemi-supervised EM algorithm to impute outcomes for historically rejected applicants — corrects selection bias in training data
MonitoringFairness metrics logged per cohort every 1000 decisions. Alert fires if parity difference > 0.15

4.4 Score Aggregation & Final Decision

FinalScore = (0.40 × dt_score) + (0.35 × ml_score) + (0.25 × fairness_adjusted_score)

if   fraud_prob > 0.75:   outcome = DECLINE   # hard block regardless of score
elif FinalScore >= 700:   outcome = APPROVE
elif FinalScore >= 550:   outcome = REFER      # manual review queue
else:                     outcome = DECLINE

Note: Weights (0.40 / 0.35 / 0.25) are configurable via the admin API without redeployment. The Decision Tree weight is deliberately high to ensure regulatory explainability requirements are met.

4.5 Airflow Retraining Pipeline (Weekly DAG)

  1. PSI Check — compare feature distributions of last 7 days vs baseline. PSI > 0.2 on any top feature triggers immediate retrain.
  2. Data Prep — fetch last 90 days of decisions with known outcomes. Run feature engineering. Split 80/10/10 train/val/test.
  3. Train — retrain LightGBM, GNN, and fairness layer as parallel Airflow tasks.
  4. Evaluate — AUC-ROC, F1, demographic parity on holdout. Gate: all metrics must beat current champion.
  5. Shadow Mode — new model scores all live traffic for 24 hours without affecting decisions. Metrics compared to champion.
  6. Promote — if shadow metrics pass, swap model artifact in S3 and hot-reload gRPC service without restart.

10.2 Interview Q&A

"Tell me about a complex DSA problem you solved"
Walk through the Interval Tree: why a plain sorted list doesn't work for overlapping ranges, how the augmented BST with maxHigh solves the stabbing query in O(log n + k), and how it gives a clean O(n log n) bulk-load at startup for 1000 lenders.
"How did you handle the ML explainability requirement?"
Dual scoring: the Decision Tree provides the regulatory-grade explanation path and SHAP values at each leaf. LightGBM provides accuracy. The two are blended. Regulators can read: "Your DTI of 0.38 reduced your score by 42 points" — that comes from the Decision Tree's SHAP output, not the LightGBM black-box.
"How does your system scale to 10K req/min?"
Kafka partitioned by applicant_id means stateful stream processing without coordination — add consumer pods and throughput scales linearly. Java API is stateless behind k8s HPA. ONNX in-process removes the model server network hop. Interval Tree and RBTree are in-memory so routing adds <5ms.
"How did you handle bias in the ML model?"
AIF360 demographic parity check at training time. If parity difference between any two cohorts exceeds 10%, the pipeline applies reweighting and adversarial debiasing before retraining. Reject inference via semi-supervised EM fixes the selection bias in historical data where rejected applicants have no outcome labels.
"What would you do differently at 10× scale?"
Replace the in-memory Interval Tree with Hazelcast distributed cache so lender rules are shared across all JVM instances. Add a feature store (Feast) to share feature computation between online scoring and offline training. Move the GNN to a dedicated GPU inference server instead of CPU-based async processing.

JPA Specification for dynamic filters — GET /applications accepts optional params (status, from, to, channel). ApplicationSpec builds the Predicate dynamically rather than requiring N overloaded query methods.

// ApplicationSpec.java
package com.lendiq.repository.spec;

import com.lendiq.dto.request.ApplicationFilterRequest;
import com.lendiq.entity.Application;
import org.springframework.data.jpa.domain.Specification;
import jakarta.persistence.criteria.*;
import java.util.ArrayList;
import java.util.List;

public class ApplicationSpec {

    public static Specification<Application> withFilters(ApplicationFilterRequest f) {
        return (root, query, cb) -> {
            List<Predicate> predicates = new ArrayList<>();

            if (f.getStatus() != null)
                predicates.add(cb.equal(root.get("status"), f.getStatus()));
            if (f.getFrom() != null)
                predicates.add(cb.greaterThanOrEqualTo(root.get("createdAt"), f.getFrom()));
            if (f.getTo() != null)
                predicates.add(cb.lessThanOrEqualTo(root.get("createdAt"), f.getTo()));
            if (f.getChannel() != null)
                predicates.add(cb.equal(root.get("sourceChannel"), f.getChannel()));
            if (f.getApplicantId() != null)
                predicates.add(cb.equal(root.get("applicantId"), f.getApplicantId()));

            return cb.and(predicates.toArray(new Predicate[0]));
        };
    }
}

package com.lendiq.exception;

import io.opentelemetry.api.trace.Span;
import org.springframework.http.*;
import org.springframework.web.bind.annotation.*;
import java.time.Instant;

@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ErrorResponse handleNotFound(ResourceNotFoundException ex) {
        return new ErrorResponse(ex.getErrorCode(), ex.getMessage(), traceId(), Instant.now());
    }

    @ExceptionHandler(DuplicateApplicationException.class)
    @ResponseStatus(HttpStatus.CONFLICT)
    public ErrorResponse handleDuplicate(DuplicateApplicationException ex) {
        return new ErrorResponse(ex.getErrorCode(), ex.getMessage(), traceId(), Instant.now());
    }

    @ExceptionHandler(VelocityFraudException.class)
    @ResponseStatus(HttpStatus.TOO_MANY_REQUESTS)
    public ErrorResponse handleVelocity(VelocityFraudException ex) {
        // Intentionally vague — do not confirm velocity details to potential fraudsters
        return new ErrorResponse("VELOCITY_EXCEEDED", "Rate limit exceeded", traceId(), Instant.now());
    }

    @ExceptionHandler(InsufficientPermissionException.class)
    @ResponseStatus(HttpStatus.FORBIDDEN)
    public ErrorResponse handleForbidden(InsufficientPermissionException ex) {
        return new ErrorResponse("FORBIDDEN", "Access denied", traceId(), Instant.now());
    }

    @ExceptionHandler(ScoringServiceUnavailableException.class)
    @ResponseStatus(HttpStatus.SERVICE_UNAVAILABLE)
    public ErrorResponse handleScoringDown(ScoringServiceUnavailableException ex) {
        return new ErrorResponse("SCORING_UNAVAILABLE",
            "Scoring service temporarily unavailable — please retry", traceId(), Instant.now());
    }

    // @Valid validation failures on request body
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ErrorResponse handleValidation(MethodArgumentNotValidException ex) {
        String msg = ex.getBindingResult().getFieldErrors().stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.joining("; "));
        return new ErrorResponse("VALIDATION_FAILED", msg, traceId(), Instant.now());
    }

    // Catch-all — never expose exception class name or internal message
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ErrorResponse handleGeneric(Exception ex) {
        log.error("Unhandled exception [traceId={}]", traceId(), ex);
        return new ErrorResponse("INTERNAL_ERROR", "An unexpected error occurred", traceId(), Instant.now());
    }

    private String traceId() {
        return Span.current().getSpanContext().getTraceId();
    }
}

package com.lendiq.config;

import com.lendiq.security.*;
import org.springframework.context.annotation.*;
import org.springframework.security.config.annotation.web.builders.HttpSecurity;
import org.springframework.security.web.SecurityFilterChain;
import static org.springframework.http.HttpMethod.*;
import static org.springframework.security.config.http.SessionCreationPolicy.STATELESS;

@Configuration
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http,
                                           JwtAuthFilter jwtFilter,
                                           ApiKeyAuthFilter apiKeyFilter,
                                           AdminIpFilter adminIpFilter) throws Exception {
        http
            .csrf(csrf -> csrf.disable())             // stateless API — CSRF not applicable
            .sessionManagement(sm -> sm.sessionCreationPolicy(STATELESS))
            .addFilterBefore(apiKeyFilter, UsernamePasswordAuthenticationFilter.class)
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
            .addFilterBefore(adminIpFilter, JwtAuthFilter.class)
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/v1/health").permitAll()
                .requestMatchers(POST, "/api/v1/applications").hasAnyRole("APPLICANT", "ADMIN")
                .requestMatchers(GET,  "/api/v1/applications/**").hasAnyRole("APPLICANT", "ADMIN")
                .requestMatchers("/api/v1/lenders/**").hasAnyRole("LENDER", "ADMIN")
                .requestMatchers("/api/v1/audit/**").hasAnyRole("ADMIN", "REGULATOR")
                .requestMatchers("/api/v1/fraud/**").hasRole("ADMIN")
                .requestMatchers("/api/v1/models/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            );
        return http.build();
    }

16.3 JwtAuthFilter

@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    @Value("${lendiq.security.jwt-secret}") private String secret;

    @Override
    protected void doFilterInternal(HttpServletRequest req,
                                    HttpServletResponse res,
                                    FilterChain chain) throws ServletException, IOException {
        String header = req.getHeader("Authorization");
        if (header == null || !header.startsWith("Bearer ")) {
            chain.doFilter(req, res); return;
        }
        try {
            String token = header.substring(7);
            Claims claims = Jwts.parserBuilder()
                .setSigningKey(Keys.hmacShaKeyFor(secret.getBytes()))
                .build()
                .parseClaimsJws(token)
                .getBody();

            List<String> roles = claims.get("roles", List.class);
            UsernamePasswordAuthenticationToken auth =
                new UsernamePasswordAuthenticationToken(
                    claims.getSubject(), null,
                    roles.stream().map(SimpleGrantedAuthority::new).collect(toList())
                );
            SecurityContextHolder.getContext().setAuthentication(auth);
        } catch (JwtException e) {
            res.sendError(HttpServletResponse.SC_UNAUTHORIZED, "Invalid or expired token");
            return;
        }
        chain.doFilter(req, res);
    }
}

16.4 AdminIpFilter

// Applied to /api/v1/models/** and /api/v1/fraud/**
// Rejects requests not originating from the internal VPN CIDRs
@Component
public class AdminIpFilter extends OncePerRequestFilter {

    @Value("${lendiq.security.admin-cidrs}") private List<String> allowedCidrs;

    private static final List<String> ADMIN_PATHS = List.of("/api/v1/models", "/api/v1/fraud");

    @Override
    protected void doFilterInternal(HttpServletRequest req,
                                    HttpServletResponse res,
                                    FilterChain chain) throws ServletException, IOException {
        boolean isAdminPath = ADMIN_PATHS.stream().anyMatch(req.getRequestURI()::startsWith);
        if (!isAdminPath) { chain.doFilter(req, res); return; }

        String callerIp = getClientIp(req);    // respects X-Forwarded-For
        if (allowedCidrs.stream().noneMatch(cidr -> IpAddressMatcher.of(cidr).matches(callerIp))) {
            res.sendError(HttpServletResponse.SC_FORBIDDEN, "IP not in admin allowlist");
            return;
        }
        chain.doFilter(req, res);
    }
}

17.7 ApplicationEventProducer

package com.lendiq.kafka.producer;

import com.lendiq.kafka.event.LoanApplicationEvent;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.stereotype.Component;
import org.springframework.beans.factory.annotation.Value;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;

@Slf4j
@Component
@RequiredArgsConstructor
public class ApplicationEventProducer {

    private final KafkaTemplate<String, LoanApplicationEvent> kafkaTemplate;

    @Value("${kafka.topic.loan-apps}") private String topic;

    public void publish(LoanApplicationEvent event) {
        // Partition key = applicant_id — ensures same applicant hits same consumer
        kafkaTemplate.send(topic, event.applicantId().toString(), event)
            .whenComplete((result, ex) -> {
                if (ex != null) {
                    log.error("Failed to publish application event [id={}]", event.applicationId(), ex);
                    throw new ScoringServiceUnavailableException("Kafka publish failed");
                }
                log.info("Published loan-app event [id={}, partition={}, offset={}]",
                    event.applicationId(),
                    result.getRecordMetadata().partition(),
                    result.getRecordMetadata().offset());
            });
    }
}
